PROBLEM SOLVED:
  - Original contract supported only flat-rate subscriptions
  - Modern SaaS platforms need multiple pricing tiers
  - Subscribers need flexible tier changes
  - Providers need tiered business models

SOLUTION PROVIDED:

  - Subscribers can upgrade/downgrade between tiers

  - Complete audit trail of tier assignments

  ✓ Provide simple, intuitive APIs for developers

================================================================================
API REFERENCE
================================================================================

PUBLIC FUNCTIONS:

  create-tier(tier-name, tier-price)
  → (ok tier-id) | (err ERR_INVALID_TIER_PRICE)
  Purpose: Create new subscription tier
  Caller: Provider
  Example: (create-tier "Pro" u2500) → tier-id u2

  change-subscription-tier(subscription-id, new-tier-id)
  → (ok new-price) | (err error-code)
  Purpose: Switch subscription to different tier
  Caller: Subscriber
  Example: (change-subscription-tier 42 u2) → ok u2500

READ-ONLY FUNCTIONS:

  get-tier(provider, tier-id)
  → {name, price, created-at} | none
  Purpose: Retrieve tier details
  Example: (get-tier provider-address u2)

  get-subscription-tier(subscription-id)
  → tier-id
  Purpose: Get current tier for subscription
  Example: (get-subscription-tier 42) → u2
