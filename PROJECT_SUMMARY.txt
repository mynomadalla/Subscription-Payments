Name: Subscription Tier Management with Dynamic Pricing

Description:
  Enables providers to create multiple subscription tiers with different pricing
  levels. Subscribers can seamlessly upgrade or downgrade between tiers, with
  the system automatically adjusting payment amounts for the next billing cycle.

Value Proposition:
  ✓ Enable tiered pricing models (Basic, Pro, Enterprise)
  ✓ Allow mid-cycle tier changes without friction
  ✓ Support modern SaaS subscription business models
  ✓ Provide simple, intuitive APIs for developers

API REFERENCE
================================================================================

Public Functions:

  create-tier(tier-name, tier-price)
  → (ok tier-id) | (err ERR_INVALID_TIER_PRICE)
  Purpose: Create new subscription tier
  Caller: Provider

  change-subscription-tier(subscription-id, new-tier-id)
  → (ok new-price) | (err error-code)
  Purpose: Switch subscription to different tier
  Caller: Subscriber

Read-Only Functions:

  get-tier(provider, tier-id)
  → {name, price, created-at} | none
  Purpose: Retrieve tier details

  get-subscription-tier(subscription-id)
  → tier-id
  Purpose: Get current tier for subscription
